# 主结果与主回归的正文逻辑

## 使用场景

用于决定主回归表、核心事实图或主结果段如何进入中文正文。表格版式交给 `econ-table-figure-design`；本文件只判断论证位置和解释顺序。

本文件中的 `表图定位`、`该结果如何支撑机制或后续检验` 等检查语言只用于作者备忘、审稿报告、任务 README 或修改清单，不得原样进入论文正文。写入正文时，应改成具体的估计结果、经济含义、识别可信度、机制连接或下一步检验对象。

## 主结果准入

正文主结果必须满足：

- 直接回答研究问题，而不是展示所有做过的规格。
- 估计设定与理论机制、识别设计一致。
- 样本、变量和固定效应的变化能够解释。
- 结果方向、量级和经济含义清楚。
- 不依赖单个偶然显著列支撑全文结论。

不满足这些条件的结果，优先放入附录、稳健性说明或规格审计文件。

## 解释顺序

主结果段默认按以下顺序写：

```text
表图定位 -> 核心系数方向和量级 -> 经济含义 -> 识别可信度或关键控制 -> 与机制的连接 -> 边界或下一步检验
```

写作时避免逐列复述。正文只解释读者理解论点所需的列：

- 哪一列是基准规格。
- 哪一列加入关键控制或固定效应。
- 哪一列用于说明样本或识别稳健性。
- 哪一列不应在正文中过度解释。

## 样本与规格审计

主表进入正文前应回答：

- 各列样本量为什么变化。
- 样本变化是否来自控制变量缺失、固定效应、单例剔除、变量口径或样本构造。
- 剩余样本是否仍代表目标识别总体。
- 规格变化是在增强可信度，还是只是追求显著性。

## 反面写法

- “列(1)-(6)均显著，因此结论稳健。”
- “加入控制变量后结果依然显著”，但没有说明控制了什么和为什么。
- 把每个系数逐一读出，却没有解释经济含义。
- 只说显著性，不给方向、量级或变量单位。

## 检查标准

主结果段完成后，读者应能知道：

1. 主结论是什么。
2. 量级意味着什么。
3. 为什么该规格可信。
4. 该结果如何支撑机制或后续检验。

上述第 4 项是作者检查问题，不是正文句式。正文中不要写“该结果支撑机制”，而要说明结果与哪条机制链条一致，或它为后续哪个机制/稳健性检验提供依据。
