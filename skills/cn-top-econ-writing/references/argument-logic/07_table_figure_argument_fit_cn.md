# 表图与论证主线的适配

## 使用场景

用于判断表格和图形是否进入正文、如何服务引言和结果叙事。版式、配色、三线表、图注和导出质量仍由 `econ-table-figure-design` 负责。

本文件中的 `主线功能`、`该结果如何支持主线`、`是否可移入附录` 等标签只用于作者备忘、审稿报告、任务 README 或修改清单，不得原样进入论文正文、表注、图注或脚注。写入正文时，应改成具体的证据角色，例如核心事实、主结果、机制证据、异质性边界、关键稳健性或附录补充。

## 正文准入原则

每张正文表图必须回答一个明确问题：

- 它是否展示核心事实、主结果、机制证据、异质性边界或关键稳健性。
- 它是否能在正文中被一两句话解释清楚。
- 它是否减少读者理解成本，而不是增加额外支线。
- 它是否与摘要、引言或理论机制中的某个承诺相对应。

不能回答上述问题的表图，应移入附录、任务审计或删除。

## 表图位置

- 引言图：适合展示核心事实、政策变化、制度场景、趋势或空间分布；不适合塞入复杂稳健性。
- 主结果表：应承接研究问题和识别设计。
- 机制表图：应承接理论机制，而不是展示所有可能渠道。
- 异质性表图：应说明边界或政策对象。
- 稳健性表：正文只保留最关键威胁，其余进入附录。

## 正文解释要求

正文引用表图时，应说明：

```text
这张表图回答了什么问题；
读者应关注哪一行、哪一列或哪个变化；
该结果如何支持主线；
是否存在必要限定。
```

不要写成“表 X 报告了结果”后就结束。表图不是附件索引，而是论证证据。

也不要写成“该表服务主线”“该图支撑主线”。应直接说读者应从表图中看到什么经济事实、估计结果、机制证据、边界条件或稳健性证据。

## 主线适配检查

```text
表图名称：
主线功能：
正文位置：
需要读者关注的核心信息：
是否与机制或识别相连：
是否可移入附录：
是否需要 econ-table-figure-design 处理版式：
```

## 常见问题

- 把工作底稿式表格放入正文。
- 一张表承担多个互不相关的功能。
- 引言图只好看，但没有支持研究问题。
- 稳健性表太多，挤压主结果和机制。
- 图表标题和正文解释不一致。
