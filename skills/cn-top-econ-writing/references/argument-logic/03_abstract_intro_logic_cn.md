# 中文摘要与引言的论证顺序

## 使用场景

用于压缩、重写或审查中文摘要和引言。目标是更快进入问题，同时不误删机制、次贡献、量级或识别设计。

## 摘要顺序

默认顺序：

```text
研究对象/问题 -> 数据或制度场景 -> 方法或识别路径 -> 主结果和量级 -> 机制或边界 -> 贡献或含义
```

执行规则：

- 目标期刊有明确字数要求时服从期刊要求；否则不要把“短”理解为机械删掉机制和第二贡献。
- 单一因果问题可以更短；混合理论-实证、结构性、机制贡献明显或多贡献论文，可以适度增加一句机制或边界说明。
- 每一句必须承担不同功能；重复背景、泛泛意义和空洞贡献优先删除。
- 摘要中至少出现一个具体方向、量级、对象或制度场景，否则容易变成空泛概述。

## 引言顺序

中文顶刊引言前半部分应尽快完成四件事：

1. 说明研究问题和现实或理论张力。
2. 交代本文如何识别、测度或论证。
3. 提前呈现主结果的方向和经济含义。
4. 说明机制及其为什么超过一个简单的数据事实。

后半部分再进入文献定位、贡献展开和结构安排。文献定位应服务贡献，不应打断主线。

## 多贡献稿件保护规则

- 有理论和实证双重贡献时，引言不能被改成纯测度论文。
- 有机制贡献时，至少保留一个段落或清楚句群说明机制链条。
- 有数据或识别贡献时，必须让读者知道可信度来自哪里。
- 有重要边界时，边界应以限定语或边界段落出现，而不是在正文中消失。

## 压缩优先级

先删：

- 宏大背景和政策口号。
- 与研究问题距离很远的文献铺垫。
- 重复说明同一个结果的句子。
- 不能指向机制、识别或贡献的泛泛意义句。

后动：

- 主贡献。
- 机制链条。
- 主结果方向和量级。
- 数据、样本、制度场景或识别来源。
- 必要边界。

## 检查标准

改写摘要或引言后回答：

```text
研究问题是否更早出现：
主结果是否仍然具体：
机制是否被保留：
第二贡献是否被保留或合理降级：
识别/数据特征是否仍然可见：
是否出现过度承诺：
```
