# 机制、异质性与稳健性的分工

## 使用场景

用于决定机制、异质性和稳健性哪些进正文、哪些进附录，以及它们在论文主线中的排序。

## 三类检验的功能

- 机制检验回答“为什么会出现主结果”。
- 异质性检验回答“结果在什么条件、主体或场景下更强或更弱”。
- 稳健性检验回答“主结果是否依赖某个口径、样本、模型或识别假设”。

不要把三类检验混写成“进一步分析”。每一小节开头应先说明它承担的论证功能。

## 机制检验

正文机制检验应满足：

- 来自理论或制度分析，而不是事后寻找显著变量。
- 能解释主结果方向，而不只是和主变量相关。
- 至少有一个可观察的中间行为、约束或资源配置变化。
- 机制结果和主结果的样本、时间和变量口径能够衔接。

如果机制证据较弱，应写成“支持机制解释”或“与该机制一致”，不要写成已经完全证明。

## 异质性检验

异质性进入正文的条件：

- 能说明机制边界、政策对象或理论条件。
- 分组依据事前可解释，而不是事后按显著性挑选。
- 分组结果与机制链条方向一致。
- 不会把主线打散成许多并列小结论。

若异质性只是补充描述，放入附录并在正文用一句话概括。

## 稳健性检验

稳健性应按疑虑组织，而不是按表格编号堆砌：

- 变量口径疑虑。
- 样本选择疑虑。
- 模型设定疑虑。
- 固定效应或趋势疑虑。
- 反向因果、遗漏变量或安慰剂疑虑。
- 聚类、权重或极端值处理疑虑。

正文只保留最能回应拒稿风险的稳健性。其余放入附录或补充材料。

## 推荐排序

默认排序：

```text
主结果 -> 识别可信度补充 -> 机制 -> 异质性 -> 稳健性摘要或附录指引
```

如果期刊或稿件习惯要求先稳健性，也可以调整为：

```text
主结果 -> 稳健性 -> 机制 -> 异质性
```

但无论如何，机制、异质性和稳健性都必须回扣主线，而不是成为独立结果清单。

## 检查标准

每个小节结尾回答：

```text
它支撑了主线的哪一环：
是否需要进入正文：
是否存在过度解释：
能否用更少表图表达：
是否应移入附录：
```
