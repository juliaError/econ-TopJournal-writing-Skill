# 理论与机制部分的论证逻辑

## 使用场景

用于写制度背景、理论分析、机制假说、研究假说和可检验命题。目标是让理论机制服务后续实证或论证，而不是独立堆概念。

## 机制链条

机制段应尽量写成：

```text
制度或市场环境 -> 经济主体 -> 约束/激励变化 -> 行为调整 -> 可观察结果 -> 可检验命题
```

检查时问：

- 主体是谁：企业、居民、政府、金融机构、平台、官员或其他主体。
- 约束是什么：预算、信息、考核、融资、监管、竞争或匹配约束。
- 行为是什么：投资、就业、创新、出口、借贷、治理、配置或迁移。
- 结果是什么：哪个变量或事实应发生何种方向变化。
- 后续如何检验：主回归、机制检验、异质性或图形是否能承接该命题。

## 制度背景的边界

- 制度背景只保留能解释识别、机制或样本选择的内容。
- 与主结果无关的政策沿革、历史细节和机构描述应压缩或移入附录。
- 如果制度背景解释的是外生冲击或比较来源，应在进入模型前讲清楚。
- 如果制度背景解释的是机制，应直接连接主体激励，而不是停留在制度介绍。

## 假说与命题

- 每个假说必须对应后文的一个主要结果、机制检验或异质性检验。
- 不设置无法检验、无法识别或只是重复主结果的假说。
- 假说的方向应来自机制链条，而不是来自“预期会显著”的经验判断。
- 理论命题应说明成立条件；边界条件不是削弱，而是提高可信度。

## 理论-实证衔接

理论或机制写完后，应能自然引出：

```text
因此，后文首先检验 [主关系]，再从 [机制]、[异质性] 和 [稳健性] 三个方面说明上述解释是否成立。
```

这个衔接句只是逻辑占位，不应机械套用；实际写作必须替换成真实变量、主体和检验。

## 常见问题

- 只有概念，没有经济主体。
- 只有政策背景，没有行为机制。
- 只有可能渠道，没有可观察含义。
- 机制和主结果方向不一致。
- 假说数量过多，正文无法逐一兑现。
