# 中文论文贡献结构诊断

## 使用场景

用于改写摘要、引言、结论、贡献段和退修回应前，判断哪些内容必须保留。压缩时不能因为追求短而删掉论文的核心贡献。

## 贡献类型

先把稿件贡献分成以下几类：

- 主贡献：论文最想让读者记住的学术增量。
- 次贡献：不是标题级贡献，但在引言、摘要或结论中被反复强调的增量。
- 机制贡献：说明某个结果为什么发生，或提出新的经济机制。
- 数据或测度贡献：构造了新指标、新数据、长期序列或更精确口径。
- 识别或论证贡献：提供了更可信的比较、冲击、制度场景或论证方式。
- 理论贡献：提出新命题、修正既有机制，或把制度事实转化为可检验假说。
- 政策或方法贡献：提供可操作的政策含义、评估框架或方法改进。

## 贡献保留规则

- 不要删除中心贡献。空间不足时压缩措辞，不压缩贡献本身。
- 如果次贡献支撑论文定位，至少在摘要、引言或结论中保留一个清楚位置。
- 如果机制本身构成贡献，不要把论文改写成只有测度或显著性结果的文章。
- 如果识别或数据是可信度来源，不能只保留结论而删掉数据或设计特征。
- 如果结论有边界条件，不能为了显得更强而删掉必要限定。

## 诊断模板

```text
论文类型：
主贡献：
次贡献：
必须保留的机制：
必须保留的数据/测度/识别特征：
必须保留的量级或事实：
必须保留的边界条件：
可压缩内容：
可移动到附录或脚注的内容：
```

## 反面风险

- 把“发现了什么”写清楚了，却删掉“为什么重要”。
- 把机制删掉，只剩政策或变量之间的相关关系。
- 把次贡献删掉，使论文看起来只是复做一个旧问题。
- 把边界删掉，使语气过强，反而削弱可信度。

## 检查标准

改写后逐项核对：

1. 主贡献是否仍然可被一句话识别。
2. 旧稿强调过的次贡献是否被合理保留或解释性删除。
3. 机制、数据、识别和量级是否至少保留其关键功能。
4. 必要边界是否还在，而不是被改成过度外推。
