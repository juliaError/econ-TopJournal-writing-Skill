# 中文论文总论证主线

## 使用场景

用于理顺全文、重构摘要和引言、压缩重复段落、判断表图是否进入正文，以及把英文或工作论文改造成中文顶刊论文。这里处理的是论证顺序，不处理中文遣词造句。

## 主线骨架

一篇中文经济学论文的主线应能压缩为：

```text
研究问题 -> 现实或理论张力 -> 机制 -> 识别或论证路径 -> 主结果 -> 经济含义 -> 贡献 -> 边界
```

检查时逐项回答：

- 研究问题：本文真正解释的经济现象是什么。
- 张力：为什么这个问题不是常识、描述或政策口号。
- 机制：哪个经济主体在何种约束下改变了什么行为。
- 识别或论证：读者凭什么相信结果来自该机制或设定。
- 主结果：方向、量级和显著性是否支撑研究问题。
- 经济含义：结果说明了哪类制度、市场或行为逻辑。
- 贡献：相对已有研究增加了什么，而不是只说“丰富”。
- 边界：结论在什么样本、时期、制度或解释条件下成立。

## 章节分工

- 摘要：给出主线的最短完整版本。
- 引言：把主线讲完整，并让主结果和机制尽早出现。
- 理论或机制：说明为什么会出现主结果，并推出可检验含义。
- 实证设计：说明识别路径、比较来源、样本和关键变量。
- 主结果：直接回答研究问题，不把正文写成回归工作日志。
- 机制检验：解释结果为什么成立，而不是替代主结果。
- 异质性：说明机制在哪里更强、更弱，或结论边界在哪里。
- 稳健性：回应最重要的识别、口径和样本疑虑。
- 结论：回到问题、贡献和边界，不新增未经检验的大结论。

## 重构规则

- 如果一个段落不能对应主线中的任何一环，优先压缩、移动或删除。
- 如果一项结果没有服务研究问题、机制或边界，不应放在正文核心位置。
- 如果摘要或引言只剩研究对象和结果、没有机制或贡献，应恢复机制和贡献。
- 如果理论部分只讲概念而不能导出后续检验，应改为“主体-约束-行为-结果”的链条。

## 输出格式

```text
一句话主线：
缺失环节：
应前置的信息：
应后移或删除的信息：
表图对主线的支撑：
下一步改写顺序：
```
