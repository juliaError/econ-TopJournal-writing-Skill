# 中文改写后的遗漏检查

## 使用场景

每次改写摘要、引言、结论、主结果、理论机制、机制检验、异质性或稳健性后使用。目标是避免“更简洁”变成“战略性信息缺失”。

本文件中的 `被删除或弱化的信息`、`是否中心/关键`、`处理方式`、`仍需用户补充的信息` 等标签只用于作者备忘、审稿报告、任务 README 或修改清单，不得原样进入论文正文、表注、图注或脚注。若发现信息缺失且无法从稿件确认，应询问用户或保留具体 `TODO`，不要自行补写事实。

## 检查步骤

1. 将新稿与旧稿逐段比较。
2. 列出被删掉或弱化的实质信息。
3. 判断每项信息是中心内容、次要但关键内容，还是可删除内容。
4. 对中心或关键内容，用更短形式恢复。
5. 对可删除内容，在任务说明或修改摘要中说明已压缩或移动。

## 必查项目

```text
研究问题：
主贡献：
次贡献：
机制：
理论命题：
数据/样本/制度场景：
识别或论证路径：
主结果方向和量级：
边界或因果解释限定：
表图承担的论证功能：
```

## 不同章节的重点

- 摘要：检查是否删掉机制、第二贡献、数据/识别特征或必要边界。
- 引言：检查是否把多贡献论文改窄成单一结果论文。
- 理论机制：检查是否删掉主体、约束、行为和可检验含义。
- 主结果：检查是否删掉量级、样本解释或识别可信度说明。
- 机制和异质性：检查是否删掉与理论机制的连接。
- 稳健性：检查是否删掉它回应的具体识别威胁。
- 结论：检查是否新增未被正文支持的外推，或删掉必要边界。

## 输出格式

```text
被删除或弱化的信息：
是否中心/关键：
处理方式：
恢复后的压缩表达：
仍需用户补充的信息：
```

## 通过标准

改写后的版本应更清楚、更短或更有重点，但不能缩小论文的真实学术主张。若简洁性和贡献完整性冲突，优先压缩表达方式，而不是删除中心贡献。
