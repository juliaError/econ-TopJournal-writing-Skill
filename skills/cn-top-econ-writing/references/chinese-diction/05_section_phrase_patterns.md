# 中文论文章节句法骨架

## 使用原则

句法骨架只提供结构，不提供可直接套用的空话。使用时必须替换为真实制度、变量、样本、结果和机制。

## 摘要

- 适用场景：中文内容提要、摘要首段。
- 推荐写法：对象、方法、发现、机制、含义依次出现。
- 反面写法：先说意义，再说“本文进行了研究”。
- 检查标准：摘要是否至少包含一个具体方向或量级。

```text
本文基于 [数据/场景]，考察 [X] 对 [Y] 的影响。
研究发现，[X] 使 [Y] [方向]，该结果在 [稳健性/替代口径] 下保持稳定。
机制分析表明，[机制] 是上述影响的重要渠道。
```

## 引言开头

- 适用场景：引言第一至三段。
- 推荐写法：从现实约束或学术矛盾进入问题。
- 反面写法：用大背景或口号铺垫过久。
- 检查标准：第一页是否出现研究问题和主要结果。

```text
[背景变化] 正在改变 [主体] 的 [约束/激励]。
这一变化使 [结果] 的形成机制成为需要解释的问题。
本文关注的问题是：[X] 是否以及如何影响 [Y]。
```

## 文献综述

- 适用场景：引言贡献段或单独文献节。
- 推荐写法：按问题和缺口组织，不按年份堆文献。
- 反面写法：连续罗列作者后接“但研究不足”。
- 检查标准：每组文献后是否有明确的本文位置。

```text
第一类研究主要解释 [A]，强调 [机制/证据]。
第二类研究转向 [B]，但对 [缺口] 的讨论仍不充分。
本文在 [对象/数据/识别/机制] 上补充上述研究。
```

## 理论机制

- 适用场景：机制假说、制度背景、理论分析。
- 推荐写法：主体、约束、行为、结果逐层推进。
- 反面写法：抽象概念连续出现，没有可检验结果。
- 检查标准：机制段是否能推出后续检验。

```text
在 [制度环境] 下，[主体] 面临 [约束]。
[冲击/政策] 改变了 [约束/收益]，从而影响 [行为选择]。
若该机制成立，则 [可观察结果] 应当发生 [方向] 变化。
```

## 实证设计

- 适用场景：模型设定前后。
- 推荐写法：先交代识别来源，再给模型。
- 反面写法：直接贴方程，再解释变量。
- 检查标准：读者是否知道比较来自哪里。

```text
本文利用 [政策/规则/冲击] 在 [地区/时间/行业/企业] 间形成的差异，识别 [X] 对 [Y] 的影响。
基准模型控制 [固定效应]，并将标准误聚类到 [层级]。
```

## 主结果

- 适用场景：基准回归表后的正文解释。
- 推荐写法：先方向和量级，再经济含义。
- 反面写法：逐列报数或只说显著。
- 检查标准：正文是否解释最重要的系数，而不是复述整张表。

```text
表 [X] 显示，[核心变量] 的系数为 [方向]，说明 [经济含义]。
加入 [控制/固定效应] 后，估计方向和量级保持稳定。
```

## 稳健性

- 适用场景：稳健性节、附录说明。
- 推荐写法：按威胁组织，不按表格编号堆砌。
- 反面写法：所有稳健性都写成“结果依然显著”。
- 检查标准：每项稳健性是否对应一个具体疑虑。

```text
针对 [识别威胁/变量口径/样本选择]，本文进一步进行 [检验]。
结果显示，核心结论未因 [替代设定] 发生实质变化。
```

## 结论

- 适用场景：结论和政策启示。
- 推荐写法：重申发现、边界和政策对象。
- 反面写法：扩写摘要或提出泛泛建议。
- 检查标准：政策含义是否落在具体工具和主体上。

```text
本文发现，[核心结论]。
这一结果表明，在 [条件] 下，[政策工具/制度安排] 会影响 [主体] 的 [行为/结果]。
相关政策设计应关注 [边界条件]。
```
