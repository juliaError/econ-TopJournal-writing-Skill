# 中文遣词规则覆盖政策

## 使用原则

本文件说明 `chinese-diction` 规则库的覆盖判断方式，但不公开具体论文名单、作者、题名、来源编号或逐篇抽取状态。使用这些规则时，应把它们视为从多类中文经济学写作实践中抽象出的句法、搭配和修稿规则，而不是任何单篇论文的原句模板。

公开 skill 不保存论文长原句。若需要回到原文核对，应使用本地任务目录中的私人审计文件，不要在公开规则中加入长引文或逐篇来源记录。

## 覆盖类别

中文遣词规则应来自多类文本功能的共同模式，而不是单篇风格：

- 问题提出；
- 概念定义；
- 机制推进；
- 识别说明；
- 结果解释；
- 文献定位；
- 贡献表述；
- 边界说明；
- 表图说明。

## 功能覆盖

| 句子功能 | 覆盖标准 | 使用提醒 |
| --- | --- | --- |
| 问题提出 | 能同时服务摘要首句、引言首段和研究问题句 | 避免把政策背景写成口号 |
| 概念定义 | 能区分理论概念、经验指标和变量代理 | 不把指标当概念本身 |
| 机制推进 | 每一步都有主体、行为和经济含义 | 不用连接词替代机制 |
| 识别说明 | 能说清样本、比较、变化来源和威胁 | 不只说“采用模型检验” |
| 结果解释 | 能写出方向、量级和经济含义 | 不复述星号或逐列流水账 |
| 文献定位 | 能按问题和缺口组织文献 | 不按年份堆文献 |
| 贡献表述 | 能相对于具体对象、机制、数据或识别说明增量 | 不只说“丰富了研究” |
| 边界说明 | 能界定样本内发现和外推范围 | 不把样本内发现无限外推 |
| 表图说明 | 能帮助读者理解证据 | 不写作者工作流 |

## 降级与缺失来源处理

若本地文本抽取质量不足、主文档缺失或访问受限，不得作为自动规则的核心支撑。相关记录可保留在私人审计目录，但公开规则只保留抽象后的覆盖类别和检查标准。

## 使用检查

在应用中文遣词规则前，先问四个问题：

1. 当前句子属于哪一种功能：问题、概念、机制、识别、结果、文献、贡献、边界，还是表图说明？
2. 该功能是否有多类文本共同支持，而不是来自单篇风格？
3. 当前改写是否只是借用抽象模式，而没有复制论文原句？
4. 改写后是否更像论文正文，而不是作者备忘、AI 总结或英文直译？
