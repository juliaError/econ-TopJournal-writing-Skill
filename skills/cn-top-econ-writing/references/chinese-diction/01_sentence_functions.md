# 中文经济学句子功能

## 使用原则

修改中文经济学论文时，先判断句子功能，再改措辞。不要只把句子变得“更顺”，而要让句子承担清楚的论文功能。

## 1. 问题提出

- 适用场景：摘要首句、引言开头、章节首段。
- 推荐写法：以具体经济对象为主语，说明其为何构成需要解释的问题。
- 反面写法：用“具有重要意义”“备受关注”替代真正的问题。
- 检查标准：读者能否在一句话内知道对象、矛盾和研究必要性。

稳定句法：

```text
[经济现象/制度安排] 是理解 [结果/机制/政策后果] 的重要切入点。
[背景变化] 使 [主体] 面临 [具体约束/激励变化]，由此提出 [研究问题]。
```

## 2. 概念定义

- 适用场景：理论机制、变量定义、制度背景。
- 推荐写法：先给概念边界，再说明经验代理。
- 反面写法：把理论概念、经验变量和口头短写混用。
- 检查标准：概念、变量、表格列名是否指向同一对象。

稳定句法：

```text
本文所称 [概念]，主要指 [可观察对象/制度安排]。
在经验分析中，本文以 [变量/指标] 作为 [理论概念] 的代理。
```

## 3. 机制推进

- 适用场景：理论机制、制度背景、机制检验之前。
- 推荐写法：按“激励变化 -> 行为调整 -> 结果变化”推进。
- 反面写法：直接说“形成机制闭环”“产生重要影响”。
- 检查标准：每一步是否有明确主体和动作。

稳定句法：

```text
[冲击/制度] 改变了 [主体] 的 [约束/收益/风险]，进而影响其 [行为选择]。
若 [机制] 成立，则应观察到 [可检验结果]。
```

## 4. 识别说明

- 适用场景：实证设计段、摘要方法句、引言研究设计句。
- 推荐写法：把样本、变化来源和识别对象放在同一句中。
- 反面写法：只写“本文采用某模型进行检验”。
- 检查标准：读者能否看出比较来自哪里、识别什么。

稳定句法：

```text
识别上，本文利用 [制度/政策/冲击] 所形成的 [变化来源]，比较 [处理组] 与 [对照组] 的 [结果差异]。
```

## 5. 结果解释

- 适用场景：主结果段、机制段、稳健性段。
- 推荐写法：方向、量级、经济含义分层写。
- 反面写法：只说“结果显著”“假说得到验证”。
- 检查标准：句子是否说明了结果对经济问题意味着什么。

稳定句法：

```text
估计结果显示，[X] 上升/实施后，[Y] [方向]，表明 [经济含义]。
这一结果与 [机制] 的解释相一致。
```

## 6. 文献定位

- 适用场景：引言贡献段、文献综述段。
- 推荐写法：先说已有文献解决了什么，再说仍缺什么。
- 反面写法：按年份罗列文献或只说“丰富了研究”。
- 检查标准：贡献是否能落到对象、机制、数据、识别或外推范围。

稳定句法：

```text
已有研究主要从 [视角] 解释 [问题]，但对 [缺口] 的讨论仍相对不足。
相较于上述文献，本文的增量在于 [具体增量]。
```

## 7. 贡献表述

- 适用场景：摘要尾句、引言贡献段、结论。
- 推荐写法：贡献必须相对于具体文献或问题。
- 反面写法：“具有重要理论意义和现实意义”单独成句。
- 检查标准：删掉“重要”后，贡献是否仍然成立。

稳定句法：

```text
本文从 [新对象/新机制/新数据/新识别] 出发，补充了关于 [文献问题] 的证据。
```

## 8. 边界说明

- 适用场景：机制、异质性、结论、政策含义。
- 推荐写法：把条件说清，不把结果无限外推。
- 反面写法：“普遍适用”“充分证明”“全面揭示”。
- 检查标准：是否写明样本、时期、制度环境或机制条件。

稳定句法：

```text
这一解释主要适用于 [样本/时期/制度条件]，其外推仍取决于 [边界条件]。
```

## 9. 表图说明

- 适用场景：表注、图注、正文引表句。
- 推荐写法：说明表图展示什么，不重复完整回归设定。
- 反面写法：把作者工作流、删列原因或内部判断写进正文。
- 检查标准：读者是否能独立理解因变量、样本、控制项和标准误。

稳定句法：

```text
表 [X] 报告 [样本] 下 [核心关系] 的估计结果。
图 [X] 展示 [变量/残差/事件窗口] 在 [分组] 间的变化。
```
