# 中文期刊投稿体例适配工作流

## 何时读取

当用户指定目标中文期刊，或要求“按某期刊投稿要求调整”“投稿前检查”“体例适配”“期刊格式调整”时，先读取本文件，再读取对应期刊文件。

## 规则优先级

按以下顺序执行，前一层优先于后一层：

1. 目标期刊官网硬性投稿要求。
2. 用户明确确认过的投稿实操流程。
3. 目标期刊已发表论文中的稳定体例。
4. `common_chinese_journal_rules.md` 中的中文期刊共同规则。
5. `cn-top-econ-writing` 的一般中文顶刊写作规则。

如果规则冲突，保留更高优先级规则，并在输出中说明冲突点。不要用样例风格覆盖官网硬性要求。

## 目标期刊选择

- 《经济学（季刊）》：读取 `economics_quarterly.md`。
- 《经济研究》《管理世界》《中国工业经济》：先读取 `journal-notes.md` 的索引；若后续有独立文件，再读取对应独立文件。
- 目标期刊不明：只使用共同规则和保守中文顶刊规则，不擅自套用某一期刊细则。

## 工作顺序

1. 明确稿件阶段：初投稿、修改稿、录用后排版、发表前校对。
2. 明确交付物：匿名 PDF、Word 文件、封面页、正文、在线附录、数据代码说明、图表文件。
3. 先做硬性合规检查：匿名、摘要、关键词、JEL、标题层级、篇幅、参考文献、图表编号、附录。
4. 再做表图准入：正文图表是否服务主线，背景图、平行趋势、稳健性扩展是否应进入附录。
5. 最后做语言和结构调整：贡献、机制、识别、结果解释和中文表达。

## 输出方式

投稿体例检查应按以下四类输出：

- **必须修改**：违反官网硬性要求或投稿流程的项目。
- **强烈建议**：符合期刊稳定风格、会明显降低编辑或审稿阅读成本的项目。
- **可选优化**：不影响合规，但有助于呈现质量的项目。
- **待用户确认**：涉及作者身份、投稿阶段、数据公开、是否超过篇幅等无法从稿件判断的项目。

不要把内部工作流解释写进论文正文、表注或图注。规则来源和取舍理由只写在检查报告、任务 README 或修改说明中。
