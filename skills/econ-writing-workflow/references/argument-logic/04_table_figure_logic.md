# Table And Figure Logic

Use this file before polishing prose around tables and figures.

## Admission Rule

A table or figure belongs in the main text only if it carries one of four jobs:

- establishes the setting or puzzle;
- explains the design or comparison;
- reports the central result;
- supports the core mechanism or strongest validity check.

Everything else should be appendix, audit notes, or deleted.

## Main Text Test

For each table or figure, write one sentence:

`This artifact is in the main text because it shows [needed link in the argument].`

If the sentence is vague, the artifact is not ready for the main text.

## Common Problems

- A descriptive figure is attractive but not needed for the question.
- A robustness table repeats routine variants without addressing a threat.
- A mechanism table is disconnected from the theory or conceptual chain.
- A heterogeneity table lists all possible splits rather than the splits that change interpretation.
- A table note explains draft history rather than sample, method, fixed effects, clustering, or variable definitions.

## Check

Main-text tables and figures should reduce cognitive load. If they force the reader to reconstruct the paper's logic, the design has failed.
