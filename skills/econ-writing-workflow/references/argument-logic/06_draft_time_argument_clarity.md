# Draft-Time Argument Clarity

Use this file before drafting or revising economics prose in English or Chinese when the task involves an abstract, introduction, literature positioning, research design, results, mechanism, heterogeneity, contribution, or conclusion.

The rule is prospective: apply it before writing each paper-facing paragraph. A final audit is useful, but it is only a backstop.

## Draft Gate

Before writing a paragraph, answer these questions internally. If the answer is missing from the manuscript context, ask the user or leave a concrete `TODO`; do not invent it.

1. What is the exact object of the sentence: theory object, empirical unit, variable, sample, policy, mechanism, or comparison?
2. What can the reader observe: data unit, variable construction, institutional event, model primitive, table column, figure pattern, or citation?
3. Is the sentence making an absolute claim, a relative claim, a causal claim, or a design claim?
4. Which section function is this paragraph serving: question, design, result, mechanism, contribution, boundary, or implication?

## Rules

### 1. Ground Abstract Concepts

Every abstract concept needs an observable or model-based anchor.

- Do not let terms such as boundary, network, governance, capability, matching, exposure, integration, adaptation, or protection float without an object.
- Name the observed object when possible: firm pair, subsidiary pair, patent co-application, city-year exposure, household decision, product market, local policy, model agent, equilibrium outcome, or measured variable.
- If a broad concept is necessary, define how the paper observes, measures, or identifies it before using the concept as an explanation.

### 2. State the Comparison

Relative and causal claims must make the comparison recoverable.

For applied work, specify as many of these as the sentence needs:

- treated or high-exposure group;
- comparison or low-exposure group;
- pre/post or other timing;
- outcome variable;
- unit of observation;
- identifying source of variation.

Do not write that an outcome "increases", "declines", "shifts", or "is stronger" when the reader cannot tell relative to whom, relative to when, or in which outcome.

### 3. Separate Design From Findings

Research design and method paragraphs should say what the paper tests or compares. Results paragraphs should say what the paper finds.

- In design sections, use neutral language: "tests whether", "compares", "examines", "distinguishes", "uses variation in".
- Do not pre-announce the conclusion in design language unless the paragraph is explicitly previewing results.
- In results sections, state the direction, magnitude, and interpretation of the finding once the evidence has been introduced.

### 4. Do Not Call Heterogeneity a Mechanism by Itself

A subgroup difference is mechanism evidence only if the paragraph explains why that difference follows from the proposed mechanism.

Use this chain:

1. Theory predicts mechanism `M`.
2. If `M` operates, the effect should differ by observable condition `Z` or through measurable channel `C`.
3. The estimate tests that implication.
4. The result supports, weakens, or limits the mechanism.

If step 2 is missing, present the estimate as heterogeneity or scope, not as mechanism.

### 5. Position Literature by Margin

When a paragraph discusses close literature, acknowledge proximity first and then distinguish the paper's margin.

Good structure:

- What the closest papers explain, estimate, or measure.
- What question, object, mechanism, setting, data level, or identification margin remains different.
- How the current paper advances that margin.

Do not write contribution paragraphs as a list of tasks the paper completed. Write them as advances to specific literature questions.

### 6. Keep Terms Stable

Use one name for the same theoretical object, empirical object, variable, treatment, comparison group, mechanism, and sample throughout a section.

- Do not switch terms merely to avoid repetition.
- If a variable has an English code name in equations, give the reader-facing term once and then use it consistently.
- If two terms are intentionally different, state the distinction before using both.

## Section Use

- Abstract: each sentence must carry an object, design, result, magnitude, mechanism, or contribution function.
- Introduction: introduce the observed object and comparison before asking the reader to accept a broad concept.
- Literature review: use "nearby but different" positioning instead of annotated bibliography.
- Research design: describe the test without smuggling in the finding.
- Results: report direction and economic magnitude, then explain the comparison and meaning.
- Mechanism: connect subgroup or channel evidence to a mechanism prediction.
- Conclusion: restate the claim within the paper's evidence boundary.

## Final Audit

Before finalizing a section, search for sentences with these risks:

- broad concepts without measurement or model anchors;
- relative words without comparison groups;
- design paragraphs that already reveal the result;
- mechanism claims that only report subgroup differences;
- contribution paragraphs organized as "we do X, we do Y";
- unnecessary synonym changes for the same object;
- author-facing planning language instead of paper-facing claims.

Revise these issues before polishing diction. Clear argument structure comes before elegant wording.
