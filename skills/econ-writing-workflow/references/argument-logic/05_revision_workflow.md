# Argument Revision Workflow

Use this file for full-paper revision, section restructuring, or "make the paper flow" tasks.

## Round 1: Map

Create a compact map:

- question;
- mechanism or hypothesis;
- design or model logic;
- central result;
- contribution;
- boundary.

Flag any missing link before editing prose.

## Round 2: Cut

Classify each paragraph:

- spine paragraph: keep and sharpen;
- support paragraph: keep if it defends a spine link;
- duplicate paragraph: merge or delete;
- appendix paragraph: move out of main text;
- workflow paragraph: remove from paper-facing prose.

## Round 3: Reorder

Move forward:

- research question;
- identifying comparison or model logic;
- central result;
- contribution margin.

Move later:

- institutional detail;
- data construction detail;
- robustness variations;
- secondary heterogeneity;
- caveats that only matter after the result is known.

## Round 4: Rewrite

Rewrite transitions so each section answers:

- what the reader has learned;
- why the next section is needed;
- how the next section supports the spine.

## Final Check

The paper should not read like the history of how the author worked. It should read like the shortest credible route from question to answer.
