# Robustness Tables

## Rule

Robustness tables should be organized by threat, not by the order in which the author ran specifications.

## Recommended Families

- Alternative outcome or treatment definitions.
- Alternative sample restrictions.
- Alternative fixed effects or controls.
- Placebo or falsification tests.
- Alternative timing or exposure definitions.
- Alternative estimator if the outcome or data-generating process requires it.

## Main Text Versus Appendix

- Put robustness in the main text only when it addresses the most obvious threat to identification.
- Move routine alternative controls, minor sample trims, and redundant specifications to the appendix.
- Summarize large robustness sets with one compact table if the coefficients are comparable.

## What To Avoid

- A table whose only message is "still significant."
- Robustness checks chosen because they preserve significance.
- Notes that describe the author's search process.

## Check

Each robustness column must be able to answer: which concern does this column address?
