# Mechanism Tables

## Rule

Mechanism tables should mirror the theory chain: shock or policy changes incentives, incentives change behavior, behavior changes outcomes.

## Recommended Structure

- One table can show several mechanism outcomes if they share a common interpretation.
- Use panel labels for mechanism families, such as financing, demand, productivity, entry, exit, or governance.
- Keep estimation sample and specification comparable to the main result where possible.
- When the mechanism outcome has a different unit or sample, explain that difference in the table note and text.

## What To Avoid

- Calling a significant auxiliary outcome "the mechanism" without a theory link.
- Mixing mediators, alternative outcomes, and heterogeneity variables in one unlabeled table.
- Presenting mechanism checks that cannot be mapped back to the theory section.

## Check

Each mechanism table should answer: what observable implication follows if the mechanism is true?
