# Abstract And Introduction Patterns

## Principle

Abstracts and introductions should move quickly from question to design to result to contribution. Do not make readers wait for the main finding.

## Abstract Pattern

### 1. Question Sentence

- Use when: first sentence of an abstract.
- Recommended: state what relationship or mechanism the paper studies.
- Avoid: broad motivation or "this paper aims to explore."
- Check: the sentence contains a specific X, Y, and setting.
- Original pattern: `This paper studies whether [X] affects [Y] in [setting].`

### 2. Design Sentence

- Use when: second or third abstract sentence.
- Recommended: name the data and variation in one compact sentence.
- Avoid: method labels without design content.
- Check: the source of comparison is visible.
- Original pattern: `I combine [data] with [source of variation] to compare [units].`

### 3. Finding Sentence

- Use when: abstract middle.
- Recommended: lead with the main result and include magnitude when possible.
- Avoid: "the results are significant."
- Check: result has direction and scale.
- Original pattern: `[Treatment] changes [outcome] by [amount], relative to [baseline].`

### 4. Mechanism Or Interpretation Sentence

- Use when: abstract has room after main finding.
- Recommended: state the mechanism as evidence or implication, not speculation.
- Avoid: vague lists of possible channels.
- Check: mechanism sentence names a measurable margin.
- Original pattern: `The effect is largest among [group], consistent with [mechanism].`

### 5. Contribution Sentence

- Use when: final abstract sentence if needed.
- Recommended: tie the result to a specific literature or policy margin.
- Avoid: "important implications" without content.
- Check: can the sentence survive without "important" or "novel"?
- Original pattern: `The findings add evidence on [literature] by showing how [mechanism] operates in [setting].`

## Introduction Pattern

### Paragraph 1. Economic Object Before Literature

- Use when: opening an introduction.
- Recommended: start from a fact, puzzle, institutional change, or economic friction.
- Avoid: "A large literature has studied..."
- Check: the reader sees the paper's object before seeing the literature list.
- Original pattern: `[Institution/shock/market] changes [actor]'s incentives in a way that can affect [outcome].`

### Paragraph 2. Research Question And Design

- Use when: after motivation.
- Recommended: state the question and empirical/theoretical route.
- Avoid: separate vague question and vague method paragraphs.
- Check: the paragraph answers "what is compared?" or "what does the model explain?"
- Original pattern: `I ask whether [X] affects [Y] by using [variation/model mechanism].`

### Paragraphs 3-4. Main Results Early

- Use when: introduction result preview.
- Recommended: give the main effect, mechanism result, and key robustness in prose.
- Avoid: saving all findings for the results section.
- Check: a skimming reader can recover the main result from page one.
- Original pattern: `The main estimate implies [effect size]. This magnitude is [benchmark], and it is robust to [key concern].`

### Paragraphs 5-6. Mechanism And Scope

- Use when: bridging result to interpretation.
- Recommended: explain why the result arises and where it should hold.
- Avoid: unsupported policy claims.
- Check: mechanism and scope are separate.
- Original pattern: `The pattern is consistent with [mechanism], which predicts a larger response when [condition].`

### Contribution Paragraph

- Use when: positioning relative to literature.
- Recommended: organize around margins, not names.
- Avoid: annotated bibliography.
- Check: each contribution states prior work, missing margin, and this paper's addition.
- Original pattern: `First, relative to studies of [A], this paper examines [B] using [design/data].`

### Roadmap

- Use when: final introduction paragraph.
- Recommended: keep short and substantive.
- Avoid: generic section travelogue.
- Check: roadmap mentions the paper's landmarks.
- Original pattern: `The next section describes [institution], which is central to the empirical design.`
